## 1. DESCRIPCION GENERAL

El **Database Documentation System** es una plataforma web completa que permite a desarrolladores, DBAs y equipos de ingeniería:

| Funcionalidad | Descripción |
|---|---|
| **Análisis automático de esquemas** | Sube archivos SQL, DBML, JSON, YAML, CSV, XLSX, MongoDB o texto personalizado y extrae automáticamente tablas, columnas, tipos de datos, claves primarias, foráneas, relaciones implícitas, triggers, procedimientos almacenados y vistas. |
| **Documentación con IA** | Genera documentación profesional en Markdown utilizando OpenAI (GPT-4), con descripciones detalladas de cada tabla, columna, relaciones, índices y recomendaciones de optimización. |
| **Diagramas ER visuales** | Renderiza diagramas entidad-relación en formato Mermaid (renderizado en vivo) y PNG (generado con NetworkX + Matplotlib vía el analizador Python). |
| **Conversión entre formatos** | Convierte esquemas bidireccionalmente entre **11 formatos**: MySQL, PostgreSQL, SQLite, MongoDB, JSON Schema, Prisma, GraphQL, DBML, CSV, YAML y JSON Crack. |
| **Generación de datos de prueba** | Produce datos sintéticos realistas (INSERT statements y documentos JSON) que respetan claves primarias, foráneas y orden topológico de tablas, usando la librería Faker. |
| **Detección de anomalías** | Identifica problemas de diseño: normalización deficiente, índices faltantes, tipos de datos inconsistentes, falta de claves primarias, convenciones de nombres inadecuadas. |
| **Exportación a PDF** | Convierte la documentación Markdown a PDF profesional con encabezados, pies de página, numeración de páginas, tablas formateadas y barras de progreso (jsPDF). |
| **Colaboración en la nube** | Guarda documentos en Supabase, compártelos con otros usuarios, edita colaborativamente con TinyMCE y visualiza cambios en tiempo real. |
| **Extensión VS Code** | Extensión híbrida (local + cloud) para analizar esquemas directamente desde el editor, sin salir del entorno de desarrollo. |
| **API pública V1** | Endpoints REST documentados con Swagger UI para integraciones externas y automatización. |
| **Skills Registry** | Catálogo de "skills" (conjuntos de reglas/prompts para agentes de IA) que permiten auditoría de índices, optimización SQL, seguridad de esquemas, migraciones y buenas prácticas ORM. |

### 2.2 Flujo de datos típico (análisis de un archivo SQL)

```
1. Usuario sube archivo SQL desde el frontend (usu_generar.js)
        │
2. POST /upload → server.js recibe el archivo (Multer)
        │
3. server.js identifica tipo de archivo y aplica parser nativo:
   • parseSQL() para archivos .sql
   • parseJSON() para archivos .json
   • parseXLSX() para archivos .xlsx/.xls
        │
4. Extrae estructura: tablas, columnas, tipos, PKs, FKs,
   relaciones, triggers, procedimientos, vistas
        │
5. El usuario elige el tipo de procesamiento:
   ├── "Analizar con IA" → generateDocumentation()
   │       └── Envía estructura a OpenAI API
   │       └── Recibe documentación Markdown detallada
   ├── "Analizar con Python" → POST /analyze-python
   │       └── Delega a python_analyzer/main.py
   │       └── Análisis profundo + diagrama ER + anomalías
   ├── "Convertir" → POST /convert
   │       └── schema_converter.py convierte entre formatos
   └── "Generar datos" → POST /generate-data
           └── data_generator.py genera INSERT statements
        │
6. Respuesta JSON con toda la información al frontend
        │
7. Frontend renderiza en tabs: Documentación, Diagrama ER,
   Esquema, Anomalías, Conversor, Datos de Prueba
        │
8. Usuario puede: guardar en Supabase, generar PDF,
   compartir, editar con TinyMCE
```

## 3. COMPONENTES INTERNOS (EN DETALLE)

### 3.1 Backend — `server.js` (1976 líneas)

Servidor principal construido con **Express.js**. Es el orquestador central de toda la aplicación.

#### Middleware
- **Multer** — Manejo de uploads de archivos (SQL, JSON, XLSX, CSV, TXT, DBML, YAML)
- **CORS** — Permite peticiones cross-origin desde el frontend y clientes externos
- **body-parser** — Parsing de JSON y URL-encoded payloads

#### Endpoints de la API REST

| Método | Ruta | Descripción |
|---|---|---|
| `POST` | `/upload` | Subida y análisis de archivos. Soporta `.sql`, `.json`, `.xlsx`, `.xls`, `.csv`, `.txt`, `.dbml`, `.yaml`, `.yml`. Aplica el parser nativo correspondiente y devuelve estructura extraída. |
| `POST` | `/analyze-python` | Delega el análisis al motor Python (`python_analyzer/main.py`). Ejecuta el script como proceso hijo y captura la salida JSON con análisis profundo, diagrama ER, anomalías y datos de prueba. |
| `POST` | `/generate-data` | Genera datos de prueba sintéticos para un esquema dado. Usa `python_analyzer/analyzers/data_generator.py` con soporte para Faker. |
| `POST` | `/convert` | Convierte un esquema entre formatos (MySQL→PostgreSQL, SQL→Prisma, etc.). Usa `python_analyzer/analyzers/schema_converter.py`. |
| `POST` | `/generate-documentation` | Genera documentación Markdown usando OpenAI (GPT-4). Recibe la estructura extraída y opcionalmente un prompt personalizado. |
| `POST` | `/generate-pdf` | Convierte documentación Markdown a PDF usando `pdfHelper.js` (jsPDF). |
| `POST` | `/api/v1/analyze` | **API pública V1**. Endpoint documentado con Swagger para integraciones externas. Acepta esquema en formato JSON y devuelve documentación, diagrama y análisis. |
| `POST` | `/api/external/login` | Login para aplicaciones cliente externas (Supabase Auth). |
| `POST` | `/api/external/register` | Registro para aplicaciones cliente externas. |
| `GET` | `/api/external/documentos` | Lista documentos de un usuario externo. |
| `POST` | `/api/external/autologin` | Auto-login vía token para apps externas. |
| `GET` | `/api/admin/logs` | Lista logs de auditoría (panel admin). Soporta filtros por usuario, acción, fecha. |
| `GET` | `/api/admin/metrics` | Métricas del sistema: total documentos, usuarios activos, análisis realizados, conversiones. |
| `GET` | `/api/admin/users` | Lista usuarios del sistema. |
| `POST` | `/api/admin/users` | Crea un nuevo usuario. |
| `PUT` | `/api/admin/users/:id` | Actualiza datos de un usuario. |
| `DELETE` | `/api/admin/users/:id` | Elimina un usuario. |

#### Parsers nativos (Node.js)

| Parser | Función | Capacidades |
|---|---|---|
| **SQL** | `parseSQL(content)` | Extrae CREATE TABLE, ALTER TABLE, columnas con tipos y restricciones, PRIMARY KEY, FOREIGN KEY, UNIQUE, DEFAULT, CHECK, índices, comentarios de tabla/columna. Soporta MySQL, PostgreSQL y SQLite. Usa regex como fallback cuando SQLGlot no está disponible. |
| **JSON** | `parseJSON(content)` | Soporta JSON Schema estándar, arrays de documentos (estilo MongoDB), objetos anidados, infiere tipos por valor. |
| **XLSX** | `parseXLSX(buffer)` | Procesa archivos Excel multi-hoja usando la librería `xlsx`. Cada hoja se interpreta como una tabla; la primera fila como cabeceras. |

#### Generación de documentación con IA

```javascript
generateDocumentation(schema, options)
```

- Construye un prompt detallado que incluye: nombre de cada tabla, columnas con tipos y restricciones, claves primarias y foráneas, relaciones, índices, triggers, procedimientos, vistas
- Envía el prompt a **OpenAI API** (modelo `gpt-4` o configurable)
- El prompt solicita: descripción general de la base de datos, detalle de cada tabla (propósito, columnas, tipos, restricciones, relaciones), análisis de índices, triggers y procedimientos, vistas, recomendaciones de optimización
- La respuesta se recibe en formato Markdown estructurado

### 3.2 Cliente Supabase — `supabaseClient.js` (15 líneas)

```javascript
const { createClient } = require('@supabase/supabase-js')
const supabase = createClient(process.env.SUPABASE_URL, process.env.SUPABASE_SERVICE_ROLE_KEY)
```

- Inicializa el cliente de Supabase con **service role key** (acceso administrativo total)
- Usado por `server.js` para:
  - **Autenticación**: validar sesiones, crear/eliminar usuarios
  - **Base de datos**: CRUD de documentos, logs de auditoría, métricas
  - **Storage**: almacenar documentos generados (PDF, diagramas)
  - **Realtime**: notificaciones de cambios (teórico, vía WebSocket `ws`)

### 3.3 Generador de PDF — `pdfHelper.js` (277 líneas)

Convierte documentación en formato Markdown a PDF profesional usando **jsPDF**.

- **Encabezados**: Título del documento con estilo
- **Texto formateado**: Párrafos con ajuste de línea automático
- **Tablas**: Renderizado de tablas Markdown con bordes, alineación y anchos calculados dinámicamente
- **Listas**: Soporte para listas numeradas y no numeradas
- **Barras de progreso**: Renderiza barras de progreso para puntajes de normalización
- **Pies de página**: Numeración de páginas automática
- **Metadatos**: Título, autor, fecha en el documento

### 3.4 Analizador Python — `python_analyzer/`

Motor de análisis profundo implementado en Python. Se comunica con el servidor Node.js mediante ejecución como proceso hijo (stdout JSON).

#### `main.py` (287 líneas) — CLI Orquestador

```
python main.py --file <archivo> --format <sql|nosql|dbml|csv|excel>
               --action <analyze|diagram|convert|generate-data>
               --target <formato_destino>
```

- **Argumentos**: `--file`, `--format`, `--action`, `--target`, `--output`
- Despacha la acción solicitada al analizador correspondiente
- Devuelve el resultado como JSON por stdout

#### `analyzers/sql_analyzer.py` (610 líneas) — Analizador SQL/DBML

| Capacidad | Implementación |
|---|---|
| **Parser SQL** | Usa **SQLGlot** como parser principal (soporta MySQL, PostgreSQL, SQLite, BigQuery, Snowflake, etc.). Tiene fallback a regex para entornos sin SQLGlot. |
| **Parser DBML** | Regex especializado para sintaxis DBML (`Table { }`, `Ref { }`, `Indexes { }`). |
| **Extracción** | Tablas, columnas (nombre, tipo, nullable, default, autoincrement), PKs (simples y compuestas), FKs (explícitas con `REFERENCES` e implícitas por convención `tabla_id`), índices, triggers, procedimientos almacenados, vistas. |
| **Relaciones implícitas** | Detecta relaciones por convención de nombres: columnas que terminan en `_id` y referencian una tabla con el mismo prefijo. |
| **Puntaje de normalización** | Evalúa FN1 (atomicidad), FN2 (dependencia funcional completa), FN3 (dependencia transitiva). Escala de 0-100%. |
| **Detección de anomalías** | Columnas sin tipo definido, tablas sin PK, tipos incompatibles entre FK y PK referenciada, nombres inconsistentes (camelCase vs snake_case), columnas no utilizadas en relaciones. |

#### `analyzers/nosql_analyzer.py` (683 líneas) — Analizador NoSQL

| Capacidad | Implementación |
|---|---|
| **Formatos soportados** | JSON, YAML, CSV, Excel (XLSX/XLS) |
| **Detección de estructura** | Identifica si es JSON Schema, array de documentos MongoDB, objeto con colecciones anidadas |
| **Inferencia de tipos** | Muestrea valores para inferir tipos: string, number, boolean, date, object, array, null |
| **Documentos anidados** | Analiza objetos y arrays anidados recursivamente como sub-entidades |
| **CSV multi-tabla** | Detecta múltiples tablas en un mismo CSV (separadas por líneas en blanco o cambios de cabecera) |
| **Anomalías** | Documentos con estructura inconsistente dentro de una misma colección, tipos mixtos en un mismo campo, campos con alta tasa de nulos |

#### `analyzers/schema_converter.py` (659 líneas) — Motor de Conversión

Convierte esquemas bidireccionalmente entre **11 formatos** usando un modelo intermedio normalizado.

```
Formato Origen → Modelo Intermedio → Formato Destino
```

| Formato origen/destino | Soporta |
|---|---|
| **MySQL** | CREATE TABLE con ENGINE, CHARSET, AUTO_INCREMENT |
| **PostgreSQL** | CREATE TABLE con SERIAL, BIGSERIAL, tipos nativos |
| **SQLite** | CREATE TABLE con INTEGER PRIMARY KEY AUTOINCREMENT |
| **MongoDB** | Colecciones con validación JSON Schema |
| **JSON Schema** | Estándar draft-07, $defs, required, properties |
| **Prisma** | Modelos con @id, @default, @relation, @unique |
| **GraphQL** | Tipos con ! (non-null), [ ] (listas), @deprecated |
| **DBML** | Sintaxis Table/Ref/Indexes para dbdiagram.io |
| **CSV** | Cabecera + tipos inferidos (compatible con importación) |
| **YAML** | Estructura jerárquica legible |
| **JSON Crack** | Formato visual para jsoncrack.com |

#### `analyzers/diagram_generator.py` (303 líneas) — Generador de Diagramas ER

- **Mermaid**: Genera código Mermaid `erDiagram` con entidades, atributos (tipos, PK, FK) y relaciones etiquetadas (one-to-one, one-to-many, many-to-many). Se renderiza en vivo en el frontend usando la librería Mermaid.js.
- **PNG**: Genera un grafo dirigido con **NetworkX** y lo renderiza como imagen PNG en base64 usando **Matplotlib**. Los nodos son tablas, las aristas son relaciones con etiquetas. Layout personalizado con posiciones calculadas.

#### `analyzers/data_generator.py` (490 líneas) — Generador de Datos de Prueba

- **Faker** (opcional): Si está instalado, usa Faker para generar datos realistas (nombres, emails, teléfonos, direcciones, fechas, textos, etc.)
- **Sin Faker**: Usa valores por defecto tipados (strings aleatorios, números secuenciales, fechas fijas)
- **Claves primarias**: Genera valores secuenciales (INT) o UUIDs únicos
- **Claves foráneas**: Referencia valores de PK ya generados en tablas padre, respetando el orden topológico
- **Orden topológico**: Calcula el grafo de dependencias entre tablas para generar datos en el orden correcto (primero tablas sin dependencias, luego las que referencian a las anteriores)
- **Salida dual**: Genera tanto `INSERT INTO` statements SQL como arrays de documentos JSON

### 3.6 Extensión VS Code — `extension.js` (921 líneas)

Extensión híbrida que combina capacidades locales y cloud.

#### Comandos registrados
- `db-documenter.open` — Abre el panel de documentación

#### Funcionamiento
1. Al ejecutar el comando, se crea un **Webview Panel** que carga la URL del frontend desplegado en Vercel
2. La extensión detecta el archivo activo en el editor y lo envía automáticamente a la webview
3. Para análisis locales (sin conexión), la extensión invoca directamente al analizador Python
4. Las configuraciones disponibles (`dbDocumenter.apiKey`, `dbDocumenter.serverUrl`, `dbDocumenter.autoAnalyze`) permiten personalizar el comportamiento

#### Flujo híbrido local/cloud
```
Archivo en editor → Extensión VS Code → ¿Conexión a internet?
  ├── Sí → Envía a Vercel (server.js) → Análisis cloud
  └── No  → Ejecuta python_analyzer/main.py local → Análisis offline
```

## 8. GUIA DE USO

### 8.1 Analizar un esquema SQL

1. Accede a la aplicación y haz login/registro
2. Ve a la sección **"Generar Documentación"**
3. Arrastra o selecciona un archivo SQL (`.sql`)
4. El sistema parsea automáticamente el archivo y muestra la estructura encontrada
5. Elige el tipo de análisis:
   - **"Analizar con IA"** → Documentación detallada con descripciones generadas por GPT-4
   - **"Analizar con Python"** → Análisis estructural profundo + diagrama ER + anomalías
6. Navega por los 6 tabs de resultados:
   - **Documentación**: Markdown renderizado con toda la info
   - **Diagrama ER**: Visualización Mermaid interactiva
   - **Esquema**: Estadísticas y tarjetas de tablas
   - **Anomalías**: Lista de problemas detectados
   - **Conversor**: Convierte a otro formato
   - **Datos de prueba**: INSERT statements generados

### 8.2 Convertir entre formatos

1. En el tab **Conversor**, selecciona el formato de origen y destino
2. El sistema convierte automáticamente usando el modelo intermedio
3. Copia el resultado o descárgalo

### 8.3 Generar datos de prueba

1. En el tab **Datos de Prueba**, configura la cantidad de registros por tabla
2. Activa/desactiva Faker para datos más realistas
3. Genera y copia los INSERT statements o documentos JSON

### 8.4 Usar la extensión VS Code

1. Instala el archivo `.vsix` en VS Code: `Extensions` → `...` → `Install from VSIX...`
2. Abre un archivo SQL en el editor
3. Ejecuta el comando `DB Documenter: Open` (Ctrl+Shift+P)
4. La extensión detecta el archivo activo y lo envía al analizador

### 8.5 Compartir documentos

1. Guarda un documento generado (botón "Guardar")
2. Ve a **"Mis documentos"** → botón "Compartir"
3. Busca usuarios por nombre o email
4. El documento aparecerá en su sección **"Compartidos conmigo"**

### 8.6 Usar el panel de administración

1. Requiere rol de administrador en Supabase
2. Accede a **"Admin"** → **"Dashboard"** para métricas
3. **"Logs"** para auditoría de acciones
4. **"Usuarios"** para gestión CRUD de usuarios
